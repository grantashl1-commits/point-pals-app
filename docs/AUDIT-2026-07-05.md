# PointPals — implementation audit (5 Jul 2026)

Verification of the current `main` against four prior specs: the
rewards/points/memories prompt (Sections A–G), the two memory-composer
revisions (v2 supersedes v1 on the microphone + Section F), and the
extended-family/realtime instructions. Legend: **PASS / PARTIAL / FAIL /
NOT FOUND**. Findings only — fixes land in the follow-up commits.

## A. Rewards: no voting, parent sets directly

**Extra bugs found while auditing E (not on the checklist):**
- The Post button enables for caption-/audio-only posts (`memories.tsx:125`) but `save()` bails without a photo (`:175`) — a dead Post button for exactly the posts the enable-condition promises.
- The storage RLS for the `memories` bucket requires paths to start with a household-id folder (`20260705011240:31-53` uses `(storage.foldername(name))[1]::uuid`), but the client uploads to bare `<id>.jpg` / `audio/<id>.webm` (`memories.ts:145,181`) — **live uploads are denied by policy**; every "remote" memory silently falls back to local-only.
- `AiIconPanel` is passed `useApp().household.id` inside conditionally-rendered JSX (`library.tsx:257,463`) — a React hooks-rules violation that can corrupt hook state when the panel toggles.
